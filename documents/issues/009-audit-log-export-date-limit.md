# 監査ログ: エクスポート日時フィルタ制限

## 目的
- 監査ログエクスポートの日時範囲を最大31日に制限し、大量データ取得によるメモリ/パフォーマンス問題を防ぐ。

## 依存
- `documents/issues/004-audit-log-read-export.md`

## 作業タスク
- [ ] `GET /api/admin/audit-logs/export` に `from` / `to` パラメータを必須化する
- [ ] `from` と `to` の差が31日を超える場合は 400 Bad Request を返す
- [ ] エラーメッセージを明確にする（例: 「エクスポート期間は最大31日です」）
- [ ] API_DOCS.md にパラメータ制限を記載する
- [ ] 既存テストを更新し、期間制限のテストケースを追加する

## 受入条件
- `from` / `to` の両方が必須
- 期間が31日を超える場合は 400 エラー
- 31日以内の場合は正常にエクスポートできる

## テスト
- 31日ちょうどでエクスポート成功
- 32日以上で 400 エラー
- `from` または `to` が未指定で 400 エラー

## メモ
- 実装開始時は本 issue を読み込んだ後にトピックブランチを作成すること
- 月次ダウンロードを想定した最大31日制限
