# 監査ログ: イベントタイプのドロップダウン化

## 目的
- 監査ログ一覧画面のイベントタイプフィルタをテキスト入力からドロップダウンに変更し、操作性を向上させる。

## 背景
- 現在はテキスト入力のため、ユーザーが有効なイベントタイプを知らないと入力できない
- 既知のイベントタイプをドロップダウンで選択できると UX が向上

## 作業タスク
- [ ] イベントタイプ一覧を定数として定義
- [ ] `admin_audit_logs/panel.rs` のイベントタイプフィルタを `<select>` に変更
- [ ] 「すべて」オプションを含める

## 受入条件
- イベントタイプがドロップダウンで選択可能
- 「すべて」を選択すると全イベントが表示される

## テスト
- 目視確認（ドロップダウンでイベントタイプを選択してフィルタリング）

## メモ
- イベントタイプ一覧: `attendance_clock_in`, `attendance_clock_out`, `request_create`, `request_approve` など
- 参考: `backend/src/middleware/audit_log.rs` の `classify_event` 関数
